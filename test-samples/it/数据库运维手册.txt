发布日期：2025年10月10日
归属部门：DBA 团队（基础设施中心）
审批人：赵国强（DBA 负责人）

一、概述

本手册涵盖公司生产环境数据库的日常运维操作规范，包括 MySQL 8.0 集群、PostgreSQL 16 集群和 Redis 7.x 集群。所有运维操作须在运维工单系统中登记，P0/P1 变更须经 DBA 负责人审批。

二、备份策略

1. MySQL 备份方案
   - 全量备份：每日凌晨 02:00 通过 Percona XtraBackup 8.0 执行，保留 30 天。
   - 增量备份：每 6 小时执行一次，基于上次全量备份的 LSN。
   - Binlog 归档：实时同步至对象存储（MinIO），保留 90 天。
   - 备份验证：每周三自动恢复至验证环境，校验数据完整性。

2. PostgreSQL 备份方案
   - 基础备份：每日 03:00 通过 pg_basebackup 执行，保留 14 天。
   - WAL 归档：使用 pgBackRest 持续归档至 S3 兼容存储。
   - PITR 恢复能力：支持精确到任意时间点的恢复，RPO < 1 分钟。

3. Redis 备份方案
   - RDB 快照：每日 04:00 执行 BGSAVE，保留 7 天。
   - AOF 持久化：核心业务 Redis 实例启用 AOF（everysec 模式）。

三、监控告警

监控平台：Prometheus + Grafana（Dashboard 编号 DB-001 ~ DB-015）

核心告警规则：
| 指标 | 阈值 | 级别 | 通知方式 |
|------|------|------|----------|
| CPU 使用率 | > 80% 持续 5min | P1 | 企业微信 + 电话 |
| 磁盘使用率 | > 85% | P1 | 企业微信 + 电话 |
| 慢查询数 | > 50/min | P2 | 企业微信 |
| 主从延迟 | > 5s | P1 | 企业微信 + 电话 |
| 连接数使用率 | > 90% | P0 | 电话轮询 |
| Redis 内存使用率 | > 75% | P2 | 企业微信 |

四、故障切换流程

1. MySQL 主从切换（计划内）
   a. 确认从库延迟为 0，执行 STOP SLAVE。
   b. 通过 Orchestrator 发起拓扑切换，验证新主库写入正常。
   c. 更新 ProxySQL 路由规则，将写流量指向新主库。
   d. 通知各业务方确认服务恢复，预计切换时间 < 30 秒。

2. MySQL 主库故障（非计划）
   a. Orchestrator 自动检测并在 10 秒内完成故障转移。
   b. 值班 DBA 在 5 分钟内确认切换状态，检查数据一致性。
   c. 记录故障时间线，24 小时内提交故障复盘报告。

3. Redis 集群节点故障
   a. Redis Cluster 自动完成 failover（超时 15 秒）。
   b. 值班人员确认 slot 迁移完成，补充新从节点。

五、变更窗口

- 常规变更窗口：每周二、四 22:00-24:00
- DDL 变更须使用 gh-ost 或 pt-online-schema-change 在线执行，禁止直接 ALTER TABLE。
- 变更回滚方案须在工单中预先填写。
