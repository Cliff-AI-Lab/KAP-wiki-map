### M14 #2 · ChunkHashCache LRU + PG lazy 模式
M5 #3 启动 SELECT 全表 → > 1M chunks 时 OOM + 启动慢。
- ChunkHashCache 加 OrderedDict-based LRU + size_limit（默认 100k）
- PgChunkHashCache 加 `lazy=True` 模式：
  - initialize 不预加载；aget miss 时按 chunk_id 单查 PG
  - 命中放进 LRU；超 limit 自动 evict
  - _pending dict 与 _cache 分开（evict 不丢未落盘数据）
  - flush 仅写 _pending（修旧 M5 #3 lazy 模式下污染 PG 的 bug）
- 工厂 KAP_CHUNK_HASH_LAZY=1 / KAP_CHUNK_HASH_LIMIT=N 切换
- 向后兼容：默认 lazy=False 走老行为
- **关联代码**：[`packages/rebuild/incremental_hash.py`](../../backend/packages/rebuild/incremental_hash.py)

### M14 #3 · DecisionLog/QueryLog 时序分区
M6/M7 lite 单表存所有事件，> 100M 行后慢。
- 新模块 `packages/observability/partitioning.py`：
  - `build_partition_ddl(table, year, month)` — 单月 DDL
  - `build_migration_ddl(table, ...)` — 完整迁移序列（RENAME → CREATE 分区主表 →
    月度分区 → INSERT 老数据 → 可选 DROP legacy）
  - `migrate_to_partitioned(conn, dry_run, drop_legacy)` — 执行入口
  - `ensure_partition_for_month` — 月度 cron 用
- 不在 lifespan 自动迁移（破坏性 + 慢；ops 显式触发）
- ISS-Job 配置文档加月度分区维护章节（cron `0 0 2 1 * ?`）
- **关联代码**：[`packages/observability/partitioning.py`](../../backend/packages/observability/partitioning.py) + [`docs/integration/iss-job-config.md`](../integration/iss-job-config.md)
