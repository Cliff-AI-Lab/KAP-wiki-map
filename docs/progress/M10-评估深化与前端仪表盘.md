## 全景成果

### M10 #1 · 多 K 召回曲线 + GT 自动构造（决策书 §5.3 评估深化）
- `MultiKRecallReport` + `run_multi_k_recall_eval(ks=[1,3,5,10])` 一次跑多 K（单次拉 max(ks) 个 sources，按 K 截断重算节省 qa 调用）
- `GroundTruthCandidate` + `auto_construct_ground_truth_candidates(min_useful_rate=0.8)` 从高 useful_rate query 反向建 gt 候选
- M10 lite 不直接入库：候选返回，等 SME 人审后调 add ground-truth 入库
- API：`POST /recall-eval/multi-k` + `GET /ground-truth/auto-construct`
- **关联代码**：[`packages/observability/recall_eval.py`](../../backend/packages/observability/recall_eval.py)

### M10 #2 · 监测条件 LLM 自学习 lite（决策书 §5.3 prompt 调优反哺）
- `classify_condition(proposal)` 启发式分类 4 监测条件（new_entity / solidification / split / standard_upgrade）
- `analyze_condition_health(proposals) -> dict[ConditionType, ConditionHealth]`
- 调优建议（启发式）：
  - total < 5 → "样本不足"
  - approve_rate < 30% → "建议收紧阈值或细化 prompt"
  - approve_rate ≥ 70% → "prompt 健康"
- 不实际改 prompt（须 SME 审）；仅生成调优报告
- API：`GET /condition-health[?project_id=]`
- **关联代码**：[`packages/observability/condition_health.py`](../../backend/packages/observability/condition_health.py)

### M10 #3 · 前端运营仪表盘（决策书 §5.3 全维度可视化）
- React 19 + Tailwind 6 卡片栅格布局
- `Promise.allSettled` 并行拉 dashboard / trend / condition-health 三端点，单失败不阻断其他卡片
- StatusBadge / DeltaBadge / PercentBadge 子组件复用
- 仅用 Lucide icons（禁 emoji per CLAUDE.md 全局约束）
- 路由：`/v15/manage/observability`
- 构建验证：tsc -b 无错；vite build 12.74 kB chunk
- **关联代码**：[`frontend/src/pages/v15/ObservabilityDashboard.tsx`](../../frontend/src/pages/v15/ObservabilityDashboard.tsx) + [`services/observabilityApi.ts`](../../frontend/src/services/observabilityApi.ts)
