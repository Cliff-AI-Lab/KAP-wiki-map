## 全景成果

### M8 #1 · portal 用户反馈埋点（决策书 §5.3 接 portal）
扩展 M7 #2 query_log，加 "有用 / 无用" 反馈：
- `QueryEvent` 新增 `useful: bool | None` / `feedback_note` / `feedback_at`
- `record_query_feedback` (sync) / `arecord_query_feedback` (async) 双 API
- 派生指标 `useful_rate` / `feedback_coverage` 进 aggregate_queries
- PG 持久化兼容老库（`ALTER TABLE ADD COLUMN IF NOT EXISTS`）
- API `POST /observability/queries/{query_id}/feedback`
- **关联代码**：[`packages/observability/query_log.py`](../../backend/packages/observability/query_log.py) + [`pg_query_log.py`](../../backend/packages/observability/pg_query_log.py)

### M8 #2 · 召回率评估管线（决策书 §5.3 准确率维度）
新增 `recall_eval` 函数式模块：
- `GroundTruthQuery` (gt_id / project_id / query_text / expected_doc_ids / note)
- `RecallEvalReport` (avg_recall / avg_precision / avg_f1 + per-query 详情)
- `run_recall_eval(qa_callable, project_id, version, k)` — 注入式评估器
  - 测试可 mock 假 qa；生产用 qa_engine.ask 包装
  - 单 query 异常吞掉视作 0 召回（不阻断）
- API 端点 5 新：ground-truth CRUD（SME 鉴权）+ run eval + reports list/latest
- 内存模式（M9 PG 持久化）
- **关联代码**：[`packages/observability/recall_eval.py`](../../backend/packages/observability/recall_eval.py)

### M8 #3 · 仪表盘集成 feedback + recall（决策书 §5.3 全维度运营观察）
dashboard 端点扩展，至此覆盖**全维度**：
- decisions (M6 #3) — 演化决策计数
- queries (M7 #2 + M8 #1) — 召回行为 + 用户反馈
- observations (M5 #2) — promote 后 7 天观察期摘要
- recall_eval (M8 #2) — ground truth 召回准确率（latest 摘要 + gt 集大小）
- 报告摘要不下发 details 全量（避免响应过大；明细走 /reports/{id}）
- **关联代码**：[`api/routers/observability.py`](../../backend/api/routers/observability.py) `dashboard` endpoint
