## 全景成果

### M7 #1 · DecisionLog PG 持久化（决策书 §5.3 跨重启不丢决策）
M6 #3 内存 list 在服务重启时丢失，加 PG write-through：
- `initialize_pg_decision_log(dsn)` 启动时建表 `decision_events` + 索引 + 水化最近 1000 条
- `record_decision (sync)` → `asyncio.create_task` 异步落盘（fire-and-forget）
- `arecord_decision (async)` → `await` 等待 PG 写入（强持久性）
- 接入：ontology.approve/reject_proposal + rebuild.promote/rollback 全部改 await arecord_decision
- 单连接 + asyncio.Lock 串行化（不引连接池依赖）
- **关联代码**：[`packages/observability/pg_decision_log.py`](../../backend/packages/observability/pg_decision_log.py) + [`decision_log.py`](../../backend/packages/observability/decision_log.py)

### M7 #2 · 查询召回埋点（决策书 §5.3 简单运营观察）
新增 `query_log` 函数式模块，记录 portal / qa 端点每次查询：
- `QueryEvent` 字段 query_id / project_id / user_id / query_text(200 字截断) / source_count / hit / latency_ms / occurred_at
- 派生指标 `hit_rate` / `avg_latency_ms` / `p95_latency_ms`
- 接入：`qa.ask_question` → await arecord_query（异常 try/except 不阻断主流程）
- PG 持久化模式同 #1（`pg_query_log.py` + `query_events` 表）
- API 端点 `GET /observability/queries[/aggregate]`
- **关联代码**：[`packages/observability/query_log.py`](../../backend/packages/observability/query_log.py) + [`pg_query_log.py`](../../backend/packages/observability/pg_query_log.py)

### M7 #3 · 综合运营仪表盘 API（决策书 §5.3 三源聚合）
单端点 `GET /api/v1/observability/dashboard`，前端运营看板一次拉全三源指标：
- decisions: 7 类计数 + approval_rate + promote_rollback_ratio
- queries: total / hits / hit_rate / avg_latency_ms / p95_latency_ms
- observations: 当前观察期摘要（status / alerts_count），active / alerting 全局计数
- 支持 since / until / project_id 三方过滤
- observations.items 字段精简（不下发 baseline + snapshots 全量，避免响应过大）
- **关联代码**：[`api/routers/observability.py`](../../backend/api/routers/observability.py) `dashboard` endpoint
