## 全景成果

### M9 #1 · GroundTruth + reports PG 持久化（决策书 §5.3 跨重启不丢评估）
- 启动时建表 `ground_truth_queries` + `recall_eval_reports` + 索引 + 全量水化 gt + 最近 N 条 reports
- 三个 sinks（fire-and-forget gt upsert/delete，await report insert）
- 单连接 + asyncio.Lock 串行化（不引连接池依赖）
- 环境变量 `KAP_RECALL_EVAL_PG=1` 启用；连不上时静默降级内存模式
- **关联代码**：[`packages/observability/pg_recall_eval.py`](../../backend/packages/observability/pg_recall_eval.py)

### M9 #2 · 召回率趋势 + 告警阈值（决策书 §5.3 漂移自动告警）
- `compute_recall_trend(project_id, lookback)` 计算 recall/precision/f1 三个 delta
- 阈值默认：召回率跌 > 10pp 触发 `recall_alert`；精确率同
- `check_recall_alerts_and_propagate(project_id)` 把告警追加到当前活跃 PromotionObservation
- `/recall-eval` POST 完成后自动跑趋势检查（仅当 project_id 非空）
- 新端点 `GET /recall-eval/trend?project_id=&lookback=`
- **关联代码**：[`packages/observability/recall_eval.py`](../../backend/packages/observability/recall_eval.py) `compute_recall_trend` / `check_recall_alerts_and_propagate`

### M9 #3 · 自动定时 eval-all 端点（外部 cron / ISS-Job 入口）
- `eval_all_projects(qa_callable)` 遍历所有有 ground truth 的 project，逐项跑评估 + 趋势检查
- 单 project 异常吞掉，不阻断
- 新端点 `POST /recall-eval/eval-all`（RequireRole SME）
- 设计同 M6 #2 tick-all：不接 Quartz 框架，HTTP 端点供任何 cron 调度器
- **关联代码**：[`packages/observability/recall_eval.py`](../../backend/packages/observability/recall_eval.py) `eval_all_projects`
